# Vibrdrome Repository Map

This document provides a comprehensive map of the Vibrdrome repository: its structure, build configuration, dependencies, entitlements, and source organization.

---

## 1. Project Overview

**Vibrdrome** is an iOS/macOS music player for [Navidrome](https://www.navidrome.org/) servers, communicating via the Subsonic API. It includes full CarPlay support for in-vehicle audio playback.

| Property            | Value                                              |
|---------------------|----------------------------------------------------|
| Language            | Swift 6.0                                          |
| Frameworks          | SwiftUI, SwiftData, AVFoundation, Combine, CarPlay |
| Build System        | XcodeGen (`project.yml` generates `Vibrdrome.xcodeproj`) |
| Deployment Targets  | iOS 17.0+, macOS 14.0+                             |
| Xcode Compatibility | 16.0+                                              |
| Concurrency Mode    | Strict (`complete`)                                |

## 5. Info.plist Files

### iOS -- `Vibrdrome/Info.plist`

- Standard bundle keys: identifier, executable, name, version, short version string, package type.
- **UIApplicationSceneManifest** with two scene session roles:
  - `CPTemplateApplicationSceneSessionRoleApplication` -- CarPlay scene
  - `UIWindowSceneSessionRoleApplication` -- Phone/iPad scene
- **UIBackgroundModes**: `audio`, `fetch`, `processing`

### macOS -- `Vibrdrome/Info-macOS.plist`

- Standard bundle keys only.
- No scene manifest. No background modes.

## 8. Key Architectural Patterns

### Singletons (all `@MainActor`)

| Singleton                     | Type        | Role |
|-------------------------------|-------------|------|
| `AppState.shared`             | `@Observable` | Credentials, server state, app-wide flags |
| `AudioEngine.shared`          | `@Observable` | Playback, queue management |
| `PersistenceController.shared`| Class        | SwiftData `ModelContainer` |
| `DownloadManager.shared`      | `@unchecked Sendable` | Background download management |

### Concurrency Model

- **Strict concurrency checking** (`complete`) enforced project-wide.
- Singletons are `@MainActor`-isolated.
- `DownloadManager` uses `NSLock` for thread-safe access from `URLSession` delegate callbacks.
- Async operations guard context validity (e.g., checking song ID matches after `await`).
- `@MainActor` closure isolation is inherited inside `@MainActor` class methods. For framework callbacks on background queues (e.g., `MPMediaItemArtwork` requestHandler), closure creation is moved to a free function outside the class.

### Credential Storage

- **Server URL and username**: `UserDefaults`
- **Password**: Keychain (via `KeychainAccess` library)
- Loaded at `AppState.init` to ensure availability for CarPlay cold-launch scenarios.

### Platform Branching

The codebase compiles for both iOS and macOS from a single target using `#if os(iOS)` / `#if os(macOS)` guards for:

- `.navigationBarTitleDisplayMode` (iOS only)
- `.fullScreenCover` (iOS only; `.sheet` on macOS)
- `.listStyle(.insetGrouped)` (iOS only)
- `scenePhase .background` (unreliable on macOS; uses `.inactive`)
- `NWPathMonitor` cellular settings (hidden on macOS)
- `UIImage` vs `NSImage` where needed

### Navigation

- **iOS**: `TabView` with `ScrollView` + `VStack` content (not `List`, due to iOS 26.2 scrolling issues).
- **macOS**: `NavigationSplitView` with sidebar.
- **Context menus**: Use `Button` + `navigationDestination(item:)` (not `NavigationLink`).
